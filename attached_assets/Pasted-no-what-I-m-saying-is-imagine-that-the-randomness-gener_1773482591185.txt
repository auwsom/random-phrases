Detecting a hidden signal embedded in a true random generator's output (like atmospheric noise from random.org) requires isolating subtle statistical deviations from perfect uniformity, as the signal must be weak to evade basic randomness tests. The goal is to boost signal-to-noise ratio (SNR) through noise reduction and pattern amplification without assuming the injection method.

## Detection Methods
- **Autocorrelation analysis**: Compute the lagged correlation function across the bitstream or phrase indices. A signal appears as non-zero peaks at specific lags (e.g., every 128 bits), while pure randomness decays to zero. Average over sliding windows to raise SNR by ~10-20dB via noise cancellation.
- **Frequency-domain (FFT/DSP)**: Transform the sequence to spectrum; periodic signals show spectral lines amid noise floor. Use Welch's method for variance reduction—SNR gain scales with √(window count). For phrases, map words to numeric codes (e.g., letter sum) first.
- **Run-length encoding stats**: Tally sequences of 0s/1s (or word repeats); excess long/short runs indicate bias. χ² test against geometric distribution; sensitive to low-frequency modulation.
- **Conditional entropy**: Measure H(X_{i+1} | X_i..X_{i-k}) for increasing context k. Pure random stays ~1 bit; signal drops as predictor leverages hidden state.
- **Machine learning classifiers**: Train LSTM or CNN on baseline "clean" random.org samples vs. suspected signal-bearing ones. Anomaly score > threshold flags it; cross-entropy loss quantifies SNR.

## SNR Amplification Techniques
| Technique | Mechanism | SNR Gain |
|-----------|-----------|----------|
| **Ensemble differencing** | Subtract mean of N parallel clean streams (same quota) from target: signal persists, noise √N-reduces. | 10*log10(N) dB (N=100 → 20dB) |
| **Lock-in amplification** | Assume known signal frequency/phase; multiply sequence by reference sinusoid, low-pass filter. | 20-40dB for periodic signals |
| **Optimal filtering** | Wiener/Kalman filter assuming signal model (e.g., AR(1)); adapts to noise power. | Model-dependent, up to 30dB |
| **Symbolic correlation** | For phrases: Convert to n-gram matrix, apply SVD; low-rank approx reveals repeating motifs. | 15dB via dimensionality cut |
| **Compression divergence** | Normalize compression ratio (e.g., gzip size / expected random size); signal compresses more. | 5-15dB proxy for structure |

## Practical Steps for Your Setup
1. Pull 10k+ integers/bits from random.org API.
2. Generate parallel "control" sequences without suspected signal.
3. Run autocorrelation/FFT on differences; plot for peaks.
4. If phrases involved, encode as bitstreams (e.g., ordinal rank in wordlist).

True RNGs like random.org pass Dieharder/NIST suites, so signals must be extremely subtle (e.g., <0.1% bias)—but targeted methods above recover them reliably at SNR >0dB after amplification. Start with autocorrelation: it's simple and reveals most periodic/modulated signals.
